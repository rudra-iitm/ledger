# 01 — Project Overview (Executive Summary)

> Audit date: 2026-07-19. Generated by a full-codebase audit (all ~20,350 lines of TypeScript across 194 tracked files were read). Line references are `file:line` at commit `b05a9f1`.

## What Ledger is

**Ledger** is a single-user, mobile-first, dark-only **personal-finance PWA** built as a **Next.js 15 static export** that uses **GitHub itself as the database**: user data lives as JSON files in a private `ledger-data` repository, read and written via Octokit from the browser. A small **Cloudflare Worker** (`worker/`) provides the only server-side pieces: GitHub OAuth code→token exchange, a market-price proxy (AMFI mutual funds, Yahoo Finance stocks, CoinGecko crypto, goldprice.org metals), data-less Web-Push bill reminders, and Splitwise-style shared-group sync backed by KV.

The product is unmistakably **India-centric**: ₹ default currency, `en-IN` lakh/crore number formatting, AMFI mutual-fund NAVs, UPI-prefix-aware merchant matching, an institution registry of 23 Indian banks, and a quick-add keyword table featuring swiggy/zomato/ola/rapido.

## Core capabilities (all verified working in code)

- **Transactions** — one `Expense` row models five types (`expense`, `income`, `transfer`, `cc_payment`, `investment`) with one-row double-entry posting to source/destination accounts.
- **Derived balances** — account balances are always recomputed from `openingBalance + Σ signed transaction deltas`; `openingDate` doubles as a "hard reset" cutoff.
- **Accounts** — cash/bank/credit-card/wallet/investment/other, with credit-card statements, utilization, pay-bill flow, debit-card records, reconciliation, and balance adjustment.
- **Investments** — holdings-as-accounts with live price refresh, portfolio P/L, SIP-style recurring investments, and savings goals.
- **Budgets** — overall monthly + per-category with near-limit (≥80%) and over-budget alerts.
- **Recurring engine** — daily/weekly/monthly/yearly templates for expense/income/transfer/CC-bill, materialized with full catch-up backfill; separate subscriptions model with cost normalization.
- **Bill splitting** — groups with equal/unequal/percentage splits (exact integer-paise math), minimum-transaction settlement optimization, and optional multi-user sharing via link/QR through the Worker.
- **Lend & Borrow** — per-person IOU tracking with repayments and net-position pages.
- **Spaces** — trip/project budget buckets tagging expenses.
- **Analytics / Insights / Monthly Review / Reports** — category breakdowns, MoM trends, CSV + PDF export.
- **Quick add** — natural-language one-liner parsing (`lunch 450`) with keyword category inference.
- **PWA** — offline app shell, installable, data-less push reminders (worker knows only dates, never amounts).
- **Auth** — GitHub OAuth (via Worker), GitHub PAT, Google (identity only), or on-device local mode. Backup/restore as JSON.

## Architectural identity

Strict three-layer separation, and the audit confirms it is actually enforced:

```
components/ (UI)  →  lib/store/app-store.ts (single Zustand store, the only bridge)
                  →  lib/domain/* (pure business logic, no React, no IO)
                  →  lib/storage/* (StorageAdapter: localStorage or GitHub via Octokit)
```

Only 2 of ~80 components touch `lib/storage` directly (both for attachment display helpers); all IO flows through the store. Zod schemas in `lib/domain/types.ts` are the single source of truth for every entity.

## The five highest-impact findings

1. **Offline boot destroys the session** — any `loadAll()` failure (airplane mode, GitHub 5xx) triggers `clearSession()` (`lib/store/app-store.ts:424-431`).
2. **Cross-device writes silently lose data** — the GitHub adapter resolves write conflicts by refetching the SHA and overwriting (`lib/storage/github.ts:76-83`).
3. **Silent data wipe on schema mismatch** — a Zod-invalid file loads as empty defaults, and the next write persists the wipe (`lib/storage/repository.ts:77-87`).
4. **Zero automated tests** covering the money math, settlement optimizer, balance sign matrix, or migrations.
5. **Worker is fully unauthenticated with wildcard CORS** — shared groups are capability-URL-only; anyone with a join code can rewrite the group.
